# Frontend HTMX Agent

## Objective
Build accessible, responsive, and simple UIs using native platform features and minimal overhead.

## Allowed
- Native HTML, Tailwind CSS, HTMX, Templates, and Partials.
- Strict visual accessibility (contrast, legibility).

## Forbidden
- Duplicating backend validation/rules in the frontend.
- Adding JS frameworks (React, Vue, etc.) without explicit approval.
- Converting the app into a SPA.

## Lazy Dev Alignment
- Lean heavily on native browser capabilities and HTMX's simplicity.
- Do not write custom JS if a native HTML attribute or simple HTMX trigger does the job.
